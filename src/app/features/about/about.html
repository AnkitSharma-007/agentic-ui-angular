Thesis

{{ config.appName }} is an agentic UI built on Angular and Gemini 3. The thesis is simple.

The UI itself is the agent's runtime.

Tool calls render real interactive components (flight cards, hotel cards, comparison tables, route maps) rather than text blobs the user has to parse. The model's reasoning is a first-class panel of the page. Pause, approve, and reject aren't ad-hoc modal callbacks but reactive edges in the agent graph. When the user picks one of three options the comparison table presented, that is the tool's return value. The model never needs a follow-up call.

deployed_code Architectural pillars

Four ideas that make Atlas tick.

layers 01 · State

Dual-view state

The UI consumes a stream of typed AgentEvents; the raw Content[] history with thoughtSignature blobs is preserved in lock-step so multi-turn loops keep their reasoning context.

pan_tool 02 · Control

Human-in-the-loop

Interruptive tools pause the agent loop and emit an interrupt_request; the user's approve / reject / select decision becomes the next round's functionResponse.

bolt 03 · Performance

Lazy descriptor loading

Tool components ship in their own chunks. The registry kicks off the dynamic import the moment Gemini emits a tool_call, so the chunk lands by the time the UI needs to render it.

replay 04 · Replay

Deterministic replays

Every turn can be saved to IndexedDB and played back with original timing. The dual-view state model means replays are byte-identical re-emissions, not best-effort reconstructions.
